# SageMate Go-To-Market 推进计划书

> 核心战略判断：**先做非 Obsidian 用户的完整方案，Obsidian 插件作为中期增长杠杆。**  
> SageMate 的竞品是 Obsidian/Notion 本身，而非 Obsidian 的某个 AI 插件。

---

## 一、战略概览

### 1.1 核心判断

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              战略定位                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│  SageMate 不是"Obsidian 的 AI 插件"，而是"AI 原生的第二大脑"。              │
│                                                                             │
│  如果只做插件 → 沦为功能增强件，和 Copilot/Smart Connections 卷功能        │
│  如果做完整方案 → 竞品是 Obsidian/Notion 本身，"AI 自动整理"是降维打击     │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 用户分层与优先级

| 优先级 | 用户群体 | 当前状态 | 策略 |
|--------|---------|---------|------|
| **P0** | 非 Obsidian 用户（笔记难民、阅读成瘾者、零笔记用户） | 无知识管理体系 | **主攻**：提供完整闭环的 AI 知识库 |
| **P1** | Obsidian 轻度用户（< 500 篇笔记） | 有 Vault 但维护累 | **跟进**：SageMate 作为更轻松的替代方案 |
| **P2** | Obsidian 重度用户（> 500 篇笔记） | 成熟工作流 | **增长杠杆**：插件增强，不替代 |

## 二、Phase 1：非 Obsidian 用户的完整方案（Month 1~3）

**目标**：让"没有笔记体系"的用户，用 SageMate 从零构建 AI 驱动的第二大脑，并形成付费意愿。

### 2.1 核心价值闭环

```
用户旅程（非 OB 用户）：

Day 0 ──安装──► 桌面应用，5 分钟配置完成
                  │
                  ▼
Day 1 ──摄入──► 拖拽 PDF / 贴 URL / 发微信消息
                  │     ↓
                  │   AI 自动编译
                  │     ↓
                  ▼   生成结构化 Wiki 页面
Day 3 ──探索──► 浏览概念页、实体页、来源页
                  │     ↓
                  │   自动双向链接
                  │     ↓
                  ▼   知识网络可视化
Day 7 ──问答──► 提问："我之前关于 Rust 的笔记怎么说？"
                  │     ↓
                  │   AI 综合所有相关页面回答
                  │     ↓
                  ▼   带 [[引用]] 的权威回答
Day 30 ─维护──► 健康巡检报告
                  │     ↓
                  │   "发现 3 个孤立页面、1 个矛盾"
                  │     ↓
                  ▼   一键修复或确认
```

### 2.2 Phase 1A：核心功能强化（Month 1）

**当前问题**：sagemate 功能已完整，但体验粗糙，普通用户上手困难。

| # | 任务 | 目标 | 优先级 |
|---|------|------|--------|
| 1.1 | **首次启动向导** | 图形化配置数据目录、LLM API key、选择工作语言 | P0 |
| 1.2 | **预置 conventions.md** | 提供 3 套模板（学术研究/个人成长/项目工作），降低冷启动门槛 | P0 |
| 1.3 | **示例数据包** | 首次启动可选加载示例（一篇论文 + 几篇概念页），让用户立即看到 AI 编译效果 | P1 |
| 1.4 | **Dashboard 增强** | 显示知识库增长曲线、最近活跃主题、待 review 的编译任务 | P1 |
| 1.5 | **批量 Ingest** | 支持文件夹拖拽批量上传、URL 列表批量采集 | P1 |
| 1.6 | **Wiki 阅读体验** | 页面间快速跳转、面包屑导航、相关页面推荐侧边栏 | P1 |

**验收标准**：
- [ ] 零技术背景用户能在 10 分钟内完成首次配置并看到第一个 AI 编译结果
- [ ] 示例数据加载后，Dashboard 显示 ≥ 5 个 wiki 页面、≥ 3 个自动链接

### 2.3 Phase 1B：桌面应用打包（Month 1~2）

**目标**：把当前的 CLI + 分离前端，包装成"双击即开"的桌面应用。

**技术方案**：Tauri（Rust 轻量壳）

```
交付形态：
┌─────────────────────────────────────────────┐
│  Tauri 应用壳（< 15MB）                      │
│  ├── 内嵌 WebView → 渲染 React 前端         │
│  ├── 后台 spawn Python 运行时               │
│  │     └── FastAPI + SQLite + 编译器        │
│  └── 系统托盘图标 + 菜单栏                  │
└─────────────────────────────────────────────┘
        │
        ▼
  SageMate-0.5.0-mac-arm64.dmg
  SageMate-0.5.0-mac-x64.dmg
  SageMate-0.5.0-win-x64.exe
  SageMate-0.5.0-linux.AppImage
```

| # | 任务 | 说明 | 优先级 |
|---|------|------|--------|
| 2.1 | **Tauri 项目初始化** | 新建 `sagemate-desktop/` 仓库，集成前后端 | P0 |
| 2.2 | **Python 运行时打包** | 使用 `pyoxidizer` 或 `python-build-standalone` 嵌入 Python | P0 |
| 2.3 | **前端静态资源内嵌** | `npm run build` 输出复制到 Tauri 的 `dist/` | P0 |
| 2.4 | **进程管理** | Tauri 负责启动/停止 Python 后端，端口自动分配 | P0 |
| 2.5 | **自动更新** | Sparkle（macOS）+ Tauri updater（Win/Linux） | P1 |
| 2.6 | **签名与公证** | macOS 代码签名 + 公证，Windows 代码签名证书 | P1 |

**验收标准**：
- [ ] 下载 `.dmg` 后拖拽安装，双击启动，无需命令行
- [ ] 首次启动弹出配置向导，完成即可使用
- [ ] 关闭窗口后后台驻留（系统托盘），下次点击图标秒开
- [ ] 自动更新检测到新版本后提示用户

### 2.4 Phase 1C：闭环体验打磨（Month 2~3）

**目标**：让非 OB 用户的"摄入 → 编译 → 探索 → 问答 → 维护"闭环体验达到产品级。

| # | 任务 | 说明 | 优先级 |
|---|------|------|--------|
| 3.1 | **编译进度可视化** | Ingest 面板实时显示编译阶段（parsing → reading_context → calling_llm → writing_pages） | P0 |
| 3.2 | **AI 问答增强** | 支持追问、支持限定搜索范围（只搜某分类/某时间段）、回答导出为 wiki 页面 | P0 |
| 3.3 | **Wiki 编辑器** | 内嵌 Markdown 编辑器，支持实时预览、wikilink 自动补全 | P1 |

## 三、Phase 2：Obsidian 插件作为增长杠杆（Month 4~6）

**目标**：不替代 Obsidian，而是让 Obsidian 用户成为"传播节点"，通过插件把 SageMate 推荐给更多人。

                              │

                              ▼
┌─────────────────────────────────────────────────────────────┐

## 四、技术架构演进路线

### 4.1 架构分层

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  用户接触层（多前端）                                                        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐    │
│  │ 桌面应用      │  │ Web UI       │  │ Obsidian     │  │ WeChat       │    │
│  │ (Tauri)      │  │ (React SPA)  │  │ Plugin       │  │ Bot          │    │

│         │                 │                 │                 │            │
│         └─────────────────┴─────────────────┴─────────────────┘            │
│                                    │                                        │
│                         HTTP / WebSocket                                   │
│                                    │                                        │
├────────────────────────────────────┼────────────────────────────────────────┤
│  API 层（FastAPI）                 ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  /ingest  /query  /lint  /projects  /obsidian/*  /agent/wechat      │   │

├────────────────────────────────────┼────────────────────────────────────────┤
│  核心引擎层                        ▼                                        │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐  ┌────────────┐          │
│  │ Ingest     │  │ Compiler   │  │ Store      │  │ LintEngine │          │
│  │ Service    │  │ (LLM)      │  │ (SQLite)   │  │            │          │
│  └────────────┘  └────────────┘  └────────────┘  └────────────┘          │
│                                    │                                        │
├────────────────────────────────────┼────────────────────────────────────────┤
│  数据层                            ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  data/raw/  +  data/wiki/  +  data/sagemate.db  +  .env           │   │
│  │  （本地文件系统，用户完全拥有）                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 关键架构决策

| 决策 | 选择 | 理由 |
|------|------|------|
| **桌面应用技术栈** | Tauri (Rust + WebView) | 安装包小（< 15MB）、原生性能、跨平台 |
| **后端是否拆分** | 不拆分，统一 FastAPI | 降低复杂度，Obsidian 插件和桌面应用共用同一后端 |
| **数据同步** | 用户自行解决（iCloud / Dropbox / Git） | 保持本地优先哲学，不碰用户数据隐私红线 |
| **LLM 接入** | 用户自备 API key（DashScope/DeepSeek/OpenAI） | 降低我们的运营成本，用户按需付费 |
| **Obsidian 插件通信** | HTTP Localhost | 最简单稳定，Obsidian 社区插件标准做法 |

## 五、商业模式设计

### 5.1 分层定价

| 层级 | 价格 | 目标用户 | 包含内容 |
|------|------|---------|---------|
| **Free** | ¥0 | 尝鲜者、学生 | 桌面应用基础功能、单个 Project、每月 50 次 LLM 调用（通过我们的 relay） |
| **Pro** | ¥29/月 或 ¥249/年 | 知识工作者、研究者 | 无限 Project、无限 LLM 调用（relay）、优先编译队列、云备份（可选） |
| **Team** | ¥99/人/月 | 小团队、研究机构 | 团队协作空间、共享知识库、权限管理、API 高级权限 |

### 5.2 收入预测（保守估计）

| 时间节点 | 用户量 | 付费转化率 | MRR |
|---------|--------|-----------|-----|
| Month 3（Phase 1 结束） | 500 | 3% | ¥435 |
| Month 6（Phase 2 结束） | 3,000 | 5% | ¥4,350 |
| Month 12 | 15,000 | 6% | ¥26,100 |

**关键假设**：Obsidian 插件上架后带来 2~3 倍自然增长。

### 5.3 成本结构

| 成本项 | 估算 | 说明 |
|--------|------|------|
| LLM Relay 成本 | 最大头 | 如果做 Free 用户的 LLM relay，需要补贴 token 费用 |
| 桌面应用签名 | ¥2,000~5,000/年 | Apple Developer + Windows 代码签名证书 |
| 服务器（可选） | ¥500/月 | 仅用于官网、更新服务器、optional 云备份 |
| 开发人力 | 主要成本 | 假设 1~2 人全职 |

**风险控制**：不做 LLM relay，要求用户自备 API key。这样零运营成本，但会降低免费用户的转化率。建议折中：Free  tier 每月赠送少量 relay 额度（如 ¥5 等值 token），Pro tier 无限 relay。
